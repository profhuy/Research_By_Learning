\begin{abstract}
Bug report quality directly affects developer triage efficiency, yet manually assessing whether a report contains sufficient Observed Behavior (OB), Expected Behavior (EB), and Steps to Reproduce (S2R) is time-consuming. This study investigates how reliably a large language model (GPT-4o mini) can automatically evaluate bug report reproducibility compared to expert manual review, using a dataset of 30 open-source GitHub bug reports sampled from pandas, scikit-learn, and vscode. Two human annotators independently labeled each report across the OB, EB, and S2R dimensions, achieving perfect inter-annotator agreement (Cohen's Kappa = 1.000), which served as the ground truth. The LLM's labels were then compared against this consensus using Cohen's Kappa. The model achieved an overall Kappa of 0.618 (raw agreement 84.4\%), exceeding the pre-registered threshold of 0.60. Performance varied substantially by dimension: OB classification reached perfect agreement (Kappa = 1.000), EB classification passed the threshold (Kappa = 0.695), while S2R classification fell below it (Kappa = 0.417). These results suggest GPT-4o mini can reliably automate OB and EB assessment but struggles to judge the completeness of reproduction steps, indicating that human review remains necessary for S2R evaluation in an LLM-assisted triage pipeline.
\end{abstract}
